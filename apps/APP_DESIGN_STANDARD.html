APP DESIGN STANDARD

Keys, Codes & Modes™ | Official Guidelines | Version 1.0

Purpose: This document defines the design standard for ALL Keys, Codes & Modes music theory applications. Use this as your foundation for creating new apps and updating existing ones.

Applies To: All 20+ apps, all future apps, no exceptions.

Created By: Benjamin Ryan, Santa Barbara, California

1. COLOR PALETTE

These are the ONLY colors to be used. Never deviate from these exact hex codes.

Primary Colors

Color Hex Code CSS Variable Usage
Teal #008080 --primary-teal Borders, buttons, accents
Gold #F4D03F --primary-gold Headings, highlights
Purple #8B6BA3 --primary-purple Gradients, accents
Coral #CF7A5A --primary-coral Warnings, errors
Dark BG #0f0f1e --dark-bg Backgrounds
Light Text #E8E8E8 --light-text Body text

2. TYPOGRAPHY

Font Families

  • Display Font: Cinzel (serif) - Headings, titles
  • Body Font: Montserrat (sans-serif) - Text, UI
  • Load from Google Fonts on every page
"KEYS, CODES & MODES" must ALWAYS use Gold → Teal gradient. Never solid colors.

3. APP LAYOUT STANDARD

Structure (Every App Must Follow)

  • Header: Logo + "KEYS, CODES & MODES" + Control Panel + Upgrade
  • Left Side (66%): Visual display (fretboard, piano, charts)
  • Right Side (33%): Control panel (buttons, settings, Start button)
  • Footer: 5 social links + copyright + disclosures

Responsive Behavior

  • Desktop (>768px): Side-by-side layout
  • Mobile (≤768px): Stacked (display on top, controls below)

4. HEADER SPECIFICATIONS

Required Elements

  • Logo (50px × 50px triangle/circle design)
  • "KEYS, CODES & MODES" with Gold → Teal gradient
  • "Control Panel" link → auth/offerings.html
  • "Upgrade" link → pricing.html

Styling

  • Background: Frosted glass with backdrop-filter: blur(10px)
  • Border-bottom: 3px solid #008080
  • Position: sticky (top: 0)
  • Padding: 1.5rem 2rem

6. INTERACTION STANDARDS

Auto-Play (NEW STANDARD)

  • ON CLICK: Sound plays INSTANTLY when user clicks key/note
  • NO DELAY: Zero delay between click and sound
  • VISUAL FEEDBACK: Element glows gold immediately
  • START BUTTON: Still visible but optional

Start Button

  • Always visible in control panel
  • Optional use - auto-play is default
  • Style: Large Teal gradient button
  • Text: "Start" or "Initialize Audio"

7. IMPLEMENTATION CHECKLIST

Every App Must Have:

  • ✓ Correct color palette (exact hex codes)
  • ✓ Cinzel + Montserrat fonts loaded
  • ✓ Header with logo + gradient text + 2 links
  • ✓ Layout: Display left (66%), Controls right (33%)
  • ✓ Start button visible and prominent
  • ✓ Auto-play enabled (instant sound)
  • ✓ Footer with 5 social links (correct PUBLIC URLs)
  • ✓ Copyright + location info
  • ✓ Responsive (mobile tested)
  • ✓ All animations smooth (0.3s ease)
